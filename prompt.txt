CAREERS & HIRING SYSTEM

This is a fully functional ATS built into the site. It runs automatically. No human needs to touch it for the pipeline to move.

Roles to list (all full-time, remote Europe):
1. Senior Backend Engineer — €85k–€115k
2. Senior Frontend Engineer — €80k–€110k  
3. AI/ML Engineer — €90k–€125k
4. Cybersecurity Engineer — €85k–€115k
5. DevOps / Infrastructure Engineer — €80k–€110k
6. Product Designer — €70k–€95k
7. Operations Analyst — €55k–€75k

Write proper, compelling job descriptions for each. Not template JDs. The kind that make a senior engineer stop scrolling.

APPLICATION FLOW:

1. Applicant reads job page, clicks Apply
2. Multi-step application form (name, contact, links, "why KoreLabs" 50–300 words, CV upload PDF)
3. Submit → CV stored in Supabase Storage (private bucket: cvs/) → applicant record created in DB
4. Immediate acknowledgment email via Resend — personalised using gpt-4o-mini with their name, role, and why-KoreLabs response. Warm, human, specific. Not generic.
5. Applicant gets a tracking portal link (token-based, no login): /apply/track/[token] — shows their pipeline stage live
6. At T+32 hours: assessment email sent — personalised, contains unique quiz link expiring in 48h
7. Quiz: 20 MCQ questions, 30 minutes, role-specific, 3 tiers (fundamentals, applied judgment, KoreLabs context), max 36 points. No score shown to applicant. Everyone who completes it moves forward.
8. 6–7 hours after quiz completion: interview invitation email — personalised, contains booking link
9. Native interview slot booking (/interview/book/[token]) — applicant picks from available slots, gets Zoom link from .env, confirmation email sent
10. Reminders at 24h and 1h before interview
11. After interview: hiring manager marks outcome in admin dashboard. Pass → next steps email. No automated rejection emails — hiring manager handles communication.

Talent pool: form on careers page — name, email, role interest, intro, CV upload. Stored separately. Acknowledgment email sent.

EMAIL PERSONALISATION: Every single outgoing email uses gpt-4o-mini to personalise based on applicant data. Pass their name, role, why-KoreLabs answer, and any available context. The output should read like a human wrote it for that specific person.

PIPELINE AUTOMATION: Use a pipeline_jobs queue table in Supabase. A cron endpoint (/api/cron/pipeline, secured, runs every 15 min via vercel.json) processes scheduled jobs. Handles all timed email sends automatically.
